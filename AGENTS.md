# Sorry. Games website rules

## Goal
Build a polished one-page static website for an indie game studio called "Sorry. Games".

## Brand tone
- smart
- stylish
- minimal
- slightly strange
- modern
- confident
- not corporate
- not childish
- not cheesy
- premium but playful

## Visual direction
- dark or moody palette is preferred
- strong typography
- clean spacing
- subtle motion only
- avoid overdesigned gamer cliches
- avoid loud gradients unless extremely restrained
- page should feel like a real indie studio with taste

## Technical rules
- use plain HTML, CSS, and a small amount of JavaScript only
- no framework unless absolutely necessary
- keep it fast and lightweight
- make all content easy to edit later
- use CSS variables for colors, spacing, and typography
- comment major sections clearly
- structure files simply
- optimize for Cloudflare Pages static hosting

## Content structure
Sections needed:
- Hero
- About
- Featured Project
- Community / Discord
- Devlog
- Screenshots + Clips
- Roadmap
- Ideas + Feedback
- Playtest Signups
- Bug Reports
- Footer

## Interaction rules
- tasteful hover states
- subtle reveal animations
- responsive on desktop and mobile
- clean button hierarchy
- accessible contrast and readable type sizes

## Output expectations
Create and maintain:
- index.html
- styles.css
- script.js
- README.md

Use placeholder content and placeholder image blocks where needed.
